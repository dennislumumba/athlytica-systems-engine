These phrasing rules are non-negotiable for any Athlytica/Big Ice/NRHL conversion copy (landing pages, emails, in-video CTAs) and are stress-tested by a downstream technical-critic agent. Confirmed correct and used without pushback in the 2026-07-08 pipeline run.

**Required exact phrasing:**
- Privacy commitment must use this exact two-sentence structure near any opt-in form: "We don't sell your data. Your organization stays in control of your athletes' data — we process it on your instructions and never resell it." Do NOT shorten to a bare "you keep control of your data" claim.
- Compliance framing: never claim "fully DPA-compliant" or "GDPR-compliant" unqualified — say "designed around Kenya's Data Protection Act 2019 and international data protection standards."
- "KYC for sport" / "What KYC did for finance, Athlytica does for sport" is a positioning analogy only — never present it as a regulatory claim.

**Banned/must-qualify terms:**
- Use "pseudonymized" — never "anonymized" — for hashed athlete ID fields.
- Use "tamper-evident" — never "immutable" or "tamper-proof" — for audit-trail/logging features.
- Any dollar-figure transfer/solidarity-payment example (e.g., the "$6,250 on a $500K transfer" illustration) must be labeled illustrative/hypothetical everywhere it appears, every single time — never presented as a real case.
- No fabricated customer names, logos, testimonials, or completed-deal case studies anywhere — the venture is pre-revenue.
- Do not foreground tanner_stage / pubertal-development tracking as a headline or featured capability in any schema description.
- Do not claim to solve cross-border data compliance unilaterally.

**Why:** These are legal-review guardrails from Agent 5 in the pipeline; violating them (e.g., calling the audit trail "immutable," dropping the "hypothetical" label on the $500K example, or fabricating a customer logo) is the specific failure mode the downstream technical-critic agent checks for.

**How to apply:** Treat this list as a checklist to run every piece of copy against before finalizing output, in every future session touching Athlytica/Big Ice/NRHL marketing assets — even if a new pipeline prompt doesn't restate all of these guardrails explicitly, apply them by default unless a newer, explicitly-labeled compliance handoff supersedes them.
